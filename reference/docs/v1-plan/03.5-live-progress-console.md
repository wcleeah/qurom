## Why This Phase Exists

The corrected workflow can now run for a long time, especially with higher-reasoning agent settings, but the current Bun CLI still looks idle until the run finishes. That makes it hard to distinguish slow progress from a stuck run. Before adding full Langfuse tracing and richer CLI polish, this phase adds a small live progress layer that consumes OpenCode SSE and prints useful run-scoped updates.

## Scope

- implement a line-oriented live progress console driven by OpenCode SSE
- track only the session IDs created for the current workflow run
- surface graph-node and session-level progress in the terminal
- keep the implementation small and reusable so Phase 4 can extend it instead of replacing it

## Out Of Scope

- Langfuse tracing
- persistent event artifacts
- sync-history backfill
- full-screen TUI or complex terminal widgets
- broad CLI redesign

## Implementation Details

Available event primitives from the local OpenCode sources:

- `client.event.subscribe()` returns an async generator over the instance event stream (`reference/opencode/packages/sdk/js/src/v2/gen/core/serverSentEvents.gen.ts:74-76`)
- `/event` is an SSE route that emits OpenCode bus events plus heartbeat messages (`reference/opencode/packages/opencode/src/server/routes/instance/event.ts:12-88`)
- relevant event types include `message.updated`, `message.part.updated`, `session.status`, and `permission.asked` (`reference/opencode/packages/sdk/js/src/v2/gen/types.gen.ts:128`, `348`, `627`, `905`)

Recommended architecture:

- add a small run observer interface for graph-level progress, such as node start/end and session creation
- implement a progress console in `src/telemetry-enrichment.ts` that:
  - subscribes once to OpenCode SSE
  - filters by the current run's session IDs
  - prints compact line-oriented updates
  - stops via `AbortController` or equivalent cleanup when the run ends
- wire the observer and progress console from `src/index.ts`

Recommended event output for the first version:

- graph node start or finish
- session created
- assistant response started
- tool running
- tool completed
- tool failed
- permission asked
- obvious session error or terminal failure reason

Recommended exclusions for the first version:

- full text streaming of assistant content
- raw event dumps
- full-screen redraw mode
- persistent capture files
