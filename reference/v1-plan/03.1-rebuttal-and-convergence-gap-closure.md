## Implementation Details

Control-flow requirements to close from the source plan:

- the designated drafter can challenge bad auditor findings and the relevant auditor can answer that challenge until the disagreement resolves (`IMPLEMENTATION_PLAN.md:109-110`)
- `maxRebuttalTurnsPerFinding` must bound repeated disagreement on the same finding (`IMPLEMENTATION_PLAN.md:1074-1077`)
- revision should continue only when `round < maxRounds` and the findings changed (`IMPLEMENTATION_PLAN.md:930-932`)
- stagnation should fail closed instead of looping forever (`IMPLEMENTATION_PLAN.md:1087-1090`)

Recommended graph changes:

- extend graph state with per-finding rebuttal turn tracking
- add one rebuttal-review decision point after auditor rebuttal responses
- if a finding remains disputed and is still below the rebuttal cap, let the drafter answer again for only those disputed findings
- if a finding reaches the cap, freeze it in its latest valid form and continue aggregation
- normalize unresolved findings after aggregation and compare them to the previous round's unresolved signature
- if unresolved findings did not change and the run is not approved, fail in a bounded way instead of revising again

CLI detail:

- Phase 4 will still own telemetry and operator-facing polish, but the core Bun entrypoint must already support `--topic` and `--file` because the source plan treats both as first-class workflow inputs (`IMPLEMENTATION_PLAN.md:991-999`)

## Execution Checklist

1. Extend graph state to track rebuttal turns and unresolved-finding signatures.
2. Add routing so the drafter can respond again to still-disputed findings while under the rebuttal cap.
3. Enforce `maxRebuttalTurnsPerFinding` when deciding whether a disputed finding can re-enter rebuttal.
4. Freeze capped findings in their latest valid form before aggregation.
5. Add convergence or stagnation detection for unchanged unresolved findings across rounds.
6. Fail in a bounded way when unresolved findings stop changing.
7. Add `--topic` and `--file` parsing to `src/index.ts`.
8. Update downstream phase inputs so progress and tracing can observe rebuttal-turn counts and bounded-failure reasons.
